## Overview

This project studies how easy or hard it is to crack passwords depending on two things:

1. How the password is built (common, human-modified, short complex, passphrase, random)
2. How the password is stored (MD5, SHA-1, SHA-256, salted SHA-256, bcrypt at costs 10/12/14)

We built a dataset of 150 passwords, hashed them using 7 configurations, ran John the Ripper with the RockYou wordlist to crack them, then performed extended analysis including entropy calculations, attacker throughput projections, and per-password crack time estimates.

```bash

cd ~/Desktop/password_project

# 2. Generate passwords
python3 scripts/step1_generate_passwords.py

# 3. Hash everything (bcrypt takes ~20 minutes)
python3 scripts/step2_generate_hashes.py

# 4. Run cracking attacks (requires john and rockyou.txt)
bash scripts/step3_run_john.sh ~/rockyou.txt

# 5. Generate base charts

```

## Key Findings

- All 30 common passwords (123456, password, qwerty, etc.) were cracked in under 4 seconds
- Human-modified passwords (P@ssw0rd, S3cur1ty!) were cracked 40% of the time via rule-based attacks
- Passphrases and random passwords survived every attack across all 16 attack sessions
- MD5, SHA-1, and SHA-256 all produced an identical crack rate of 28% — switching algorithms made no difference
- bcrypt(14) reduced the crack rate to 1.3% and limits an attacker to fewer than 1 guess per second
- Passphrases average 147.9 bits of entropy vs 39.3 bits for short complex passwords
- bcrypt(14) is ~61,000x slower than MD5, making offline brute-force effectively impossible for strong passwords
